## Fixing Common Studio Roadblocks

- Sanity Studio login failing? Confirm account access, project permissions, dataset selection, and environment configuration before changing application code.
- Schema not appearing? Restart the local development server, check Sanity Studio schema exports, and verify that the document type is included in the configured schema list.
- Preview not loading? Review frontend preview routes, token handling, CORS settings, and the relationship between Sanity Studio hosting and the target application.
- Plugin conflict? Disable recent Sanity Studio plugins one at a time, then check version compatibility with Sanity Studio v3 and the current package manager lockfile.
- Deployment issue? Recheck Sanity Studio deployment commands, environment variables, project identifiers, and build output so the hosted editor matches the local workspace.
